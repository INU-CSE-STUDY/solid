### ❓ 무엇을 위한 공부인가요?

객체지향 언어인 **`Java/Kotlin`** 로 **`Spring/Spring Boot`** 개발을 하지만, **절차 지향적**으로 코드를 작성하는 경우가 많다.

객체지향 언어를 사용하는 만큼, **`Spring/Spring Boot`** 개발을 할 때에도 **객체 지향적으로 구현하는 연습**을 하기 위함이다.

### ❓ 어떤 식으로 진행되나요?

간단한 상황(사칙연산 API 구현하기 등)에서 다양한 **디자인 패턴을 적용해 객체 지향적으로 API를 구현하는 방식**

### ❓ Java 언어로 해도 되지 않나요? 왜 프레임워크를 활용하나요?

순수 **`Java/Kotlin`** 언어로는 객체 지향적으로 구현할 수 있지만, **프레임워크 환경에서는 절차 지향적으로 코드를 작성하는 경우**가 많다.

또, 형식적인 객체지향 패턴을 사용하기보다는, **왜 이 패턴이 필요한지를 이해하고 사용할 수 있는 능력**을 기르기 위함이다.

# 🎯 진행 방식

## 1️⃣ Git 및 프로젝트 관리

### 📝 프로젝트 정보

| JDK | 25.0.1 (LTS) |
| --- | --- |
| Framework | Spring Boot 4.0.1 |
| Build Tool | Gradle |
| Architecture | Hexagonal architecture |

### 🗓️ 기한

- 주제가 정해지면 상황에 맞춰 마감일 지정
    - 마감일이 지나기 전에 **`Pull Request`** 가 올라와 있어야 함

### 🌳 브랜치 관리

- **참여자의 이니셜**로 브랜치 생성
- 디자인 패턴을 적용한 코드 작성 및 학습
    - **`@Profile("참여자 이니셜")`** 어노테이션 활용해서 코드 작성하기
- **`Pull Request`** 로 학습 내용과 자신의 생각을 간단히 작성
    - 사용한 **디자인 패턴 이름과 간략한 설명** 작성 필수

### 💬 코드 리뷰

- **`Pull Request`** 에 올라온 내용을 참고하여 코드 리뷰

  **❌ 코드 컨벤션 리뷰 ❌**

  **⭕ 디자인 패턴에 대한 리뷰 ⭕**


## 2️⃣ 블로그 작성

### ⚠️ 블로그 작성 전에는 다음이 완료되어야 합니다.

- **`Pull Request`** 작성 및 다른 참여자 **`Pull Request`** 코드 리뷰 완료
- **코드 리뷰에서 개선 사항이 있을 경우, 해당 내용 업데이트**

---

**`Pull Request`** 에 작성한 내용을 바탕으로 스타일에 맞게 블로그 작성하기!
